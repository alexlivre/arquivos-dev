# SEO Specialist — Landing Pages (PT-BR)

## Instructions
Siga este guia passo a passo sempre que a skill for ativada.

### 1. **Identifique a solicitação e ative a skill**
- Ative quando o usuário mencionar: otimizar página/landing para SEO, rankear para termos, criar title/meta description, definir H1/H2 e estrutura, melhorar snippet/CTR orgânico, sugerir palavras-chave e intenção de busca, criar FAQ para SEO, definir canonical/noindex/indexação.

### 2. **Avalie se a skill é apropriada (ou use com cautela)**
- **Use normalmente** para páginas que devem ranquear organicamente.
- **Use com cautela ou recomende `noindex`** para:
  - Páginas 100% para tráfego pago que não devem indexar (priorize conversão).
  - Conteúdo com claims sensíveis (saúde/finanças/jurídico) — exija texto responsável sem promessas.

### 3. **Colete os inputs necessários (se faltarem)**
Se faltar contexto, faça no máximo 6 perguntas objetivas:
   1. URL atual (se existir) e se a página deve indexar (sim/não/depende).
   2. Tipo de página (landing, vendas, home, serviços, produto, artigo, etc.).
   3. Objetivo da página (conversão/cadastro/agendamento/compra) e CTA primário.
   4. Público-alvo (persona), região/idioma, e oferta (o que é, diferencial, provas reais).
   5. Tema principal e 1–3 concorrentes/serps (se o usuário tiver).
   6. Palavras-chave desejadas (se houver) e restrições de compliance (o que não pode dizer).
- Se o usuário não souber as palavras-chave: sugira hipóteses e peça confirmação antes de finalizar o SEO Pack.

### 4. **Siga os princípios fundamentais (não-negociáveis)**
- Conteúdo útil, confiável, bem organizado e fácil de ler.
- Títulos e descrições únicos por página (evitar repetição/boilerplate).
- Clareza do “título principal” (H1) e hierarquia consistente de headings.
- Evitar keyword stuffing e repetição artificial.
- URLs descritivas e legíveis.
- Links internos com âncoras descritivas (não genéricas).
- Imagens com contexto e alt text descritivo e relevante.

### 5. **Execute o procedimento de otimização (runbook)**
   1. **Defina estratégia de indexação**:
      - Se a página for para tráfego pago e não fizer sentido ranquear: recomende `noindex`.
      - Se deve ranquear: confirme 1 intenção principal e 1 termo principal (primary keyword).
   2. **Mapeie intenção de busca e promessa**:
      - Intenção: informacional / comercial / transacional / navegacional.
      - Promessa: o que o usuário quer resolver e como a página entrega isso.
   3. **Monte o “Keyword Map” (sem exagero)**:
      - Primary keyword (1).
      - Secondary keywords (3–6).
      - Sinônimos/variações (5–10).
      - Termos proibidos (se houver compliance).
   4. **Arquitetura do conteúdo (outline)**:
      - Proponha uma estrutura com H1 único e H2s que cubram a intenção.
      - Garanta que existe um “main title” claro na primeira dobra (H1).
   5. **Meta layer (CTR e entendimento)**:
      - Crie opções de `<title>`: descritivo, conciso, único, sem repetição.
      - Crie meta description: resumo curto e específico da página, não lista de keywords.
      - Garanta consistência entre title/H1/conteúdo (mesma linguagem e promessa).
   6. **Conteúdo on-page (semântica e utilidade)**:
      - Recomende seções que respondem ao que a busca quer (ex.: “o que é”, “para quem é”, “como funciona”, “benefícios”, “provas”, “FAQ”).
      - Sugira parágrafos/bullets com clareza e especificidade.
      - Inclua “provas substitutas” honestas quando não houver números (processo, bastidores, critérios, garantias reais).
   7. **Links internos e navegação**:
      - Sugira 3–8 links internos relevantes.
      - Âncoras descritivas (ex.: “planos e preços de [serviço]”, “como funciona [produto]”).
   8. **Imagens e mídia**:
      - Sugira 3–6 alt texts prontos (descritivos, contextuais, sem stuffing).
      - Recomende posicionamento de imagens perto do texto relevante.
   9. **Duplicidade/canonical (se aplicável)**:
      - Se existir variação de URL/parametrização: recomende canonical para a URL preferida.
      - Se houver páginas muito similares: diferencie propósito ou consolide.
   10. **Checklist final**:
       - Title único? H1 único? Meta description específica? Outline cobre intenção?
       - Conteúdo está legível e bem seccionado?
       - Links internos ajudam descoberta sem competir com CTA?
       - Indexação está correta (index/noindex)?

### 6. **Entregue o SEO Pack no formato padrão**
Sempre entregue neste formato (Markdown):

**A) Diagnóstico rápido**
- Indexar? (sim/não) + motivo
- Intenção principal
- Risco principal (ex.: canibalização, conteúdo raso, duplicidade)

**B) Keyword Map**
- Primary keyword
- Secondary keywords
- Variações e sinônimos
- Termos a evitar

**C) Meta (pronto para colar)**
- 5 opções de `<title>`
- 3 opções de meta description
- 1 sugestão de slug/URL (se aplicável)

**D) Headings e outline**
- 1 H1 recomendado + 3 alternativas
- Estrutura de H2/H3 sugerida (com objetivo de cada seção)

**E) Conteúdo recomendado (por seção)**
- O que deve entrar em cada seção (bullets)
- FAQ (6–10 perguntas com respostas curtas)

**F) Links internos**
- Lista de páginas para linkar + texto-âncora sugerido

**G) Imagens**
- 3–6 sugestões de alt text (por imagem)

**H) Implementação técnica (se o usuário pedir)**
- Sugestões de canonical/noindex
- Sugestões de structured data (apenas se fizer sentido)

### 7. **Respeite as restrições (NÃO FAÇA)**
- Não invente dados, “estudos”, números, depoimentos ou prêmios.
- Não use keyword stuffing nem repita termos de forma artificial.
- Não proponha práticas enganosas (cloaking, doorway pages, texto oculto).
- Não sacrifique clareza da página por “SEO” (priorize utilidade + intenção).

## Examples

### Exemplo 1 — Otimização de landing para ranquear
**Solicitação do usuário**:  
"Quero que essa landing page ranqueie para 'software de ponto eletrônico'."

**Ações do assistente (resumo)**:
- Pergunta se deve indexar.
- Produz SEO Pack com keyword map, title/meta, H1/H2, seções e FAQ, links internos, alt text.

### Exemplo 2 — Página não deve indexar
**Solicitação do usuário**:  
"Crie uma landing para campanha paga de um webinar; não quero no Google."

**Ações do assistente (resumo)**:
- Recomenda `noindex`.
- Ainda entrega meta/title coerentes (para consistência), mas prioriza outline e conteúdo para conversão.

### Exemplo 3 — Sugestão de palavras-chave e estrutura
**Solicitação do usuário**:  
"Preciso de sugestões de palavras-chave e estrutura de H2s para minha página de serviços de limpeza residencial."

**Ações do assistente (resumo)**:
- Pergunta: tipo de página, objetivo, público, região.
- Se o usuário não souber palavras-chave, sugere hipóteses (ex.: "limpeza de casa", "faxina residencial", "diarista") e pede confirmação.
- Entrega SEO Pack com keyword map, meta, headings, outline e FAQ.
